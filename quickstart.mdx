## Overview

This guide will walk you through deploying a PyWorker instance on Vast.ai and generating your first image using the ComfyUI serverless worker.

<Steps>

<Step title="Create a Vast.ai Endpoint">

1. Visit the [ComfyUI Serverless Template](https://cloud.vast.ai/?ref_id=62897&creator_id=62897&name=ComfyUI%20(Serverless))
2. Configure your serverless endpoint with the required environment variables:

```bash
BACKEND=comfyui-json
PYWORKER_REPO=https://github.com/ByQwank/pyworker
PROVISIONING_SCRIPT=https://raw.githubusercontent.com/ByQwank/pyworker/main/default.sh
HF_TOKEN=<your-huggingface-token>
PYWORKER_MANIFEST_SECRET=<your-shared-secret>
```

3. For testing, use the default options presented by the web interface
4. Note your endpoint name (default: `my-comfyui-endpoint`)

<Note>
  Get your HuggingFace token from [https://huggingface.co/settings/tokens](https://huggingface.co/settings/tokens). You'll need it to download models.
</Note>

</Step>

<Step title="Install the Client SDK">

Clone the PyWorker repository and install dependencies:

```bash
git clone https://github.com/ByQwank/pyworker
cd pyworker
pip install uv
uv venv -p 3.12
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
uv pip install -r requirements.txt
```

Set your Vast.ai API key:

```bash
export VAST_API_KEY=<your-vast-api-key>
```

<Tip>
  Get your API key from the [Vast.ai account page](https://cloud.vast.ai/account/)
</Tip>

</Step>

<Step title="Wait for Provisioning">

The worker needs to download models and dependencies before it's ready. This typically takes 5-15 minutes depending on the template.

You can monitor provisioning status by checking the worker logs in the Vast.ai console. Look for:

- Custom nodes cloning
- Model downloads via aria2c
- Provisioning complete marker: `/workspace/.provisioning-complete`

<Warning>
  Don't send requests before provisioning completes! The worker will reject them with a health check failure.
</Warning>

</Step>

<Step title="Generate Your First Image">

Once provisioning is complete, run the test client:

<CodeGroup>

```bash Default Prompt
python -m workers.comfyui-json.client
```

```bash Custom Prompt
python -m workers.comfyui-json.client --prompt "a cat sitting on a rainbow"
```

```bash With Parameters
python -m workers.comfyui-json.client \
  --prompt "sunset over mountains" \
  --width 1024 \
  --height 1024 \
  --steps 30
```

```bash Custom Endpoint
python -m workers.comfyui-json.client \
  --endpoint my-custom-endpoint \
  --prompt "cyberpunk city at night"
```

</CodeGroup>

The client will:
1. Connect to your Vast.ai endpoint
2. Submit the generation request
3. Wait for completion
4. Download the generated image to `./generated_images/comfy_{seed}.png`

</Step>

<Step title="Verify the Output">

Check the generated image:

```bash
ls -lh generated_images/
```

You should see a PNG file with your generated image. The client output will show:

```
============================================================
COMFYUI TEXT-TO-IMAGE DEMO
============================================================
Prompt: a beautiful sunset over mountains, digital art, highly detailed
Size: 512x512, Steps: 20, Seed: 1234567890

🎨 Generating image...

✅ Generation complete!
Worker URL: https://...
  💾 Saved: generated_images/comfy_1234567890.png
```

</Step>

</Steps>

## Advanced Usage

### Using Custom Workflows

You can execute any ComfyUI workflow by providing a workflow JSON file:

```bash
python -m workers.comfyui-json.client --workflow my_workflow.json
```

The workflow JSON should be a complete ComfyUI API format workflow. Export it from ComfyUI using "Save (API Format)".

### S3 Upload Integration

Automatically upload generated images to S3-compatible storage:

```bash
# Set S3 credentials
export S3_ENDPOINT_URL="https://your-account.r2.cloudflarestorage.com"
export S3_BUCKET_NAME="my-bucket"
export S3_ACCESS_KEY_ID="your-access-key-id"
export S3_SECRET_ACCESS_KEY="your-secret-access-key"

# Run with S3 upload enabled
python -m workers.comfyui-json.client --prompt "a beautiful landscape" --s3
```

Images will be saved locally AND uploaded to `s3://{bucket}/comfyui/{filename}`.

<Note>
  Requires `boto3`: Install with `pip install boto3`
</Note>

### Client CLI Reference

| Flag | Default | Description |
|------|---------|-------------|
| `--endpoint` | `my-comfyui-endpoint` | Vast endpoint name |
| `--prompt` | Default prompt | Text prompt for image generation |
| `--workflow` | None | Path to custom workflow JSON file |
| `--width` | 512 | Image width in pixels |
| `--height` | 512 | Image height in pixels |
| `--steps` | 20 | Number of denoising steps |
| `--seed` | Random | Random seed for reproducibility |
| `--s3` | Disabled | Upload generated images to S3 |

## API Reference

### Request Format

The worker accepts requests at `/generate/sync` in two modes:

<CodeGroup>

```json Modifier Mode
{
  "input": {
    "request_id": "uuid-string",
    "modifier": "Text2Image",
    "modifications": {
      "prompt": "a beautiful landscape",
      "width": 1024,
      "height": 1024,
      "steps": 20,
      "seed": 123456789
    },
    "s3": {},
    "webhook": {}
  }
}
```

```json Custom Workflow Mode
{
  "input": {
    "request_id": "uuid-string",
    "workflow_json": {
      "3": {
        "inputs": {
          "seed": 42,
          "steps": 20,
          "cfg": 8,
          "sampler_name": "euler",
          "scheduler": "normal",
          "denoise": 1,
          "model": ["4", 0],
          "positive": ["6", 0],
          "negative": ["7", 0],
          "latent_image": ["5", 0]
        },
        "class_type": "KSampler"
      }
    }
  }
}
```

</CodeGroup>

### Health Check Endpoint

Check if the worker is ready:

```bash
curl https://your-worker-url/readyz
```

Response:
```json
{
  "ok": true,
  "healthcheck_url": "http://127.0.0.1:18288/health",
  "model_server_base_url": "http://127.0.0.1:18288"
}
```

## Troubleshooting

### Worker Not Ready

If you get health check failures:

1. Check if `/workspace/.provisioning-complete` exists on the worker
2. Verify `/workspace/.provisioning-failed` doesn't exist
3. Review worker logs for provisioning errors
4. Ensure all models downloaded successfully (check aria2c output)

### LoRA Download Failures

If LoRAs fail to download:

1. Verify `HF_TOKEN` is set correctly
2. Check HuggingFace repo permissions: `Dylaaann/Lora`
3. Ensure `PYWORKER_MANIFEST_SECRET` matches between client and worker
4. Review worker logs for HMAC signature verification errors

### Missing API Key

If you see "API key missing" errors:

```bash
# Verify your API key is set
echo $VAST_API_KEY

# Set it if missing
export VAST_API_KEY=<your-key>
```
