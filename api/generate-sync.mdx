## Endpoint

```
POST /generate/sync
```

Processes ComfyUI workflows synchronously and returns results when generation is complete. Supports both predefined modifiers (like Text2Image) and custom workflow JSON.

## Authentication

Requires a valid Vast.ai API key set in the `VAST_API_KEY` environment variable when using the Serverless SDK.

## Request Configuration

- **Parallel Requests**: Not allowed (one generation per worker at a time)
- **Queue Time**: 0 seconds (requests are rejected if worker is busy)
- **Workload**: Automatically calculated based on workflow complexity (dimensions, frames, steps)

## Request Body

Choose either **Modifier Mode** (predefined workflows) or **Custom Workflow Mode** (raw ComfyUI JSON).

## Examples

### Text-to-Image with SDK

```python
import uuid
from vastai import Serverless

async with Serverless() as client:
    endpoint = await client.get_endpoint(name="my-comfyui-endpoint")
    
    payload = {
        "input": {
            "request_id": str(uuid.uuid4()),
            "modifier": "Text2Image",
            "modifications": {
                "prompt": "a beautiful sunset over mountains, digital art",
                "width": 1024,
                "height": 1024,
                "steps": 30,
                "seed": 42
            }
        }
    }
    
    response = await endpoint.request("/generate/sync", payload, cost=100)
    print(f"Worker URL: {response['url']}")
```

### Custom Workflow with SDK

```python
import uuid
import json
from vastai import Serverless

with open("my_workflow.json", "r") as f:
    workflow_json = json.load(f)

async with Serverless() as client:
    endpoint = await client.get_endpoint(name="my-comfyui-endpoint")
    
    payload = {
        "input": {
            "request_id": str(uuid.uuid4()),
            "workflow_json": workflow_json
        }
    }
    
    response = await endpoint.request("/generate/sync", payload, cost=100)
```

### Fetching Generated Images

```python
import aiohttp

async def fetch_image(worker_url: str, filename: str):
    url = f"{worker_url}/view"
    params = {"filename": filename, "type": "output"}
    
    async with aiohttp.ClientSession() as session:
        async with session.get(url, params=params) as resp:
            if resp.status == 200:
                image_data = await resp.read()
                with open(f"output_{filename}", "wb") as f:
                    f.write(image_data)
                print(f"Saved: output_{filename}")

# Extract filename from response
filename = response["response"]["comfyui_response"]["node_id"]["outputs"]["images"][0]["filename"]
await fetch_image(response["url"], filename)
```

## Error Codes

### 400 Bad Request

**Invalid payload structure**
```json
{
  "error": "Payload must be an object."
}
```

**Missing workflow_json in custom mode**
```json
{
  "error": "payload.input must be an object."
}
```

**Invalid LoRA filename**
```json
{
  "error": "Invalid LoRA filename 'invalid-file.txt'."
}
```

**LoRA manifest validation failed**
```json
{
  "error": "lora_manifest.signature is required when PYWORKER_MANIFEST_SECRET is set."
}
```

**Manifest signature mismatch**
```json
{
  "error": "lora_manifest signature mismatch."
}
```

**Manifest expired**
```json
{
  "error": "lora_manifest signature expired."
}
```

**Unauthorized LoRA usage**
```json
{
  "error": "Workflow requested LoRAs not allowed by signed manifest: model1.safetensors, model2.safetensors"
}
```

### 503 Service Unavailable

**Worker busy (queue time = 0)**
```json
{
  "error": "Worker is currently processing another request. Please try another worker or retry later."
}
```

### 500 Internal Server Error

**LoRA download failed**
```json
{
  "error": "HF_TOKEN is required to download LoRAs."
}
```

**ComfyUI processing error**
```json
{
  "error": "ComfyUI workflow execution failed."
}
```

## LoRA Model Management

The worker automatically downloads required LoRA models from HuggingFace before processing workflows.

### Automatic Detection

LoRAs are detected from workflow nodes with inputs matching `lora_name`, `lora_1_name`, `lora_2_name`, etc.

### Manual Specification

```json
{
  "input": {
    "workflow_json": { ... },
    "required_loras": ["style-lora.safetensors", "face-lora.safetensors"]
  }
}
```

### Signed Manifests

When `PYWORKER_MANIFEST_SECRET` is configured, requests must include a signed `lora_manifest`:

```json
{
  "input": {
    "workflow_json": { ... },
    "lora_manifest": {
      "requiredLoras": ["approved-model.safetensors"],
      "generationId": "gen-12345",
      "endpoint": "direct-instance",
      "issuedAt": 1709481600000,
      "signature": "a1b2c3d4e5f6..."
    }
  }
}
```

The signature is computed as:
```python
import hmac, hashlib, json

payload = {
    "endpoint": endpoint,
    "generationId": generation_id,
    "issuedAt": int(issued_at),
    "requiredLoras": sorted(loras)
}

signature = hmac.new(
    secret.encode("utf-8"),
    json.dumps(payload, ensure_ascii=False, separators=(",", ":"), sort_keys=True).encode("utf-8"),
    hashlib.sha256
).hexdigest()
```

## Workload Calculation

The endpoint automatically calculates request cost based on:

- **Dimensions**: Width and height (512px grid units)
- **Frames**: Video length for image-to-video workflows (default: 17)
- **Steps**: Denoising steps (default: 8)
- **Modifiers**: RIFE interpolation (×1.35), upscaling (×1.25), video combines (×1.15 per extra)

Base formula:
```python
grid_w = ceil(width / 512)
grid_h = ceil(height / 512)
workload = grid_w × grid_h × frames × steps × modifiers × 0.6
```

This ensures fair resource allocation across worker instances.
